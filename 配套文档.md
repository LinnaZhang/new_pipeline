### 1.2 DataReader（core_engine/data_reader.py）

| 项目 | 说明 |
|------|------|
| **功能** | 带缓存的 Excel 数据读取器。验证文件存在性和格式有效性，读取源 Excel 工作表的全量数据并缓存，避免重复 I/O。提供按指标代码（AA 码）精确提取数据的功能。 |
| **入参** | `file_path: str` — 源 Excel 文件路径 |
| **出参** | 实例化后提供两个核心方法返回数据 |
| **可复用范围** | 所有行业通用，作为数据读取的标准入口 |
| **依赖限制** | 依赖 pandas、openpyxl；仅支持 .xlsx/.xls 格式 |

**核心方法：**

| 方法 | 功能 | 入参 | 出参 |
|------|------|------|------|
| `read_sheet_data(sheet_name)` | 读取并缓存整个工作表数据 | `sheet_name: str` | `pd.DataFrame`（无表头，全量数据） |
| `read_indicator_data(sheet_name, indicator_code)` | 读取指定工作表中特定 AA 码指标的数据 | `sheet_name: str`, `indicator_code: str` | `Dict{"metadata": Dict, "data": pd.DataFrame}` |

| 方法 | 功能说明 |
|------|---------|

| `merge_indicators(data_reader, source_sheet, indicators)` | 将多个指标合并为一个 DataFrame |

## 二、各行业特殊处理逻辑文档

### 2.1 航空行业（plugins/aviation_plugin.py）

#### 航空整体处理流程
1. 逐个航空公司 sheet 写入基础指标数据 → 计算同比上年增长率
2. 计算同比上年差值 → 计算同比2019增长率 → 计算与2019差值
3. 清理早年公式 → 调整全年行后格式 → 清除YTD 2018差值 → 清除特殊列
4. 月度/季度汇总表写入标题和跨表公式

#### 核心处理函数

**`aviation_write_airline_sheet()`**
- **功能**：写入航空公司基础指标数据。以第一个指标日期为基准，补全缺失月份，写入日期列和各指标列，支持单位转换（春秋/吉祥×0.01，国泰×0.001）。自动生成季度汇总和 YTD 统计。对国泰航空特殊处理缺失值（列H=列B-列E，列Q=列K-列N）。
- **特殊逻辑**：自动补全起始月份到首个数据月份之间的缺失日期；国泰航空数据通过已有列派生缺失列

**`aviation_apply_yoy_formulas()`**
- **功能**：计算同比上年增长率（`=当期/上年同期-1`）。根据标签类型（年/季度/月份）自动匹配偏移行数（年偏移2行，季偏移5行，月偏移12行）。
- **特殊逻辑**：识别"年"、"Q"、"2017-"等不同日期标签格式，分别计算偏移量

**`aviation_apply_yoy_diff_formulas()`**
- **功能**：计算同比上年差值（`=当期-上年同期`）。逻辑同增长率但为减法计算。

**`aviation_apply_yoy19_formulas()`**
- **功能**：计算相对于 2019 年同期的增长率（`=当期/2019同期-1`）。定位 2019 年各季度和各月份数据行，用于计算疫情前后恢复程度。
- **特殊逻辑**：分别定位 2019 年全年行、各季度行和各月份行，按标签类型匹配对应行号

**`aviation_apply_diff19_formulas()`**
- **功能**：计算相对于 2019 年同期的差值（`=当期-2019同期`）。

**`aviation_clear_early_years_data()`**
- **功能**：清除 2014-2017 年早年的同比/同比2019 列公式，保持空白。

**`aviation_adjust_format_after_full_year()`**
- **功能**：全年行以下的数据格式调整为一位小数（`0.0%`）。

**`aviation_clear_ytd_2018_diff_data()`**
- **功能**：清除 YTD 区域中 2018 年的同比差值数据。

**`aviation_clear_ax_ay_az_columns()`**
- **功能**：清除季度/YTD/全年区域中 AX、AY、AZ 三列的残留公式，防止 `#NAME?` 错误。

**`aviation_write_monthly_report_header_info()`**
- **功能**：处理月度汇总表。从各航空公司子表获取最新日期，统一后写入汇总表标题（"【申万交运】XXXX年X月航空公司数据汇总"），写入"当期值/同比上年/同比2019"参数列。

**`aviation_write_quarterly_report_header_info()`**
- **功能**：处理季度汇总表。从各航空公司子表获取最新季度标识，写入标题和"当期值/同比上年Q/同比2019Q"参数列。

**`aviation_write_report_data()`**
- **功能**：在汇总表中写入 INDIRECT 跨表引用公式，从各航空公司子表自动汇总数据到总表。

### 2.2 宏观行业（plugins/macro_plugin.py）

#### 宏观整体处理流程
1. 写入指标组数据（日期+多指标列）
2. 创建基础透视表（月-日为行，年份为列，自动处理闰年2月29日→2月28日）
3. 生成春节标签透视表（分析春节前后数据变化趋势）
4. 生成除夕相对天数透视表（分析除夕前后数据变化趋势）
5. 生成周度数据透视表（按 ISO 周统计）
6. 生成按年日期脚手架（用于构建日期框架）

#### 核心处理函数

**`macro_write_indicator_group()`**
- **功能**：写入一组指标数据到目标工作表。以第一个指标日期为基准，后续指标依次写入相邻列。
- **入参**：indicators（指标代码列表）、start_row、start_col、date_format、ascending
- **特殊逻辑**：降序排列（最新日期在前），指标值0自动转为None

**`macro_create_pivot_table()`**
- **功能**：创建基础透视表，按"月-日"为行、年份为列进行透视。
- **特殊逻辑**：自动将闰年 2月29日 映射到 2月28日；仅包含最近 N 年（默认7年）数据

**`add_global_spring_festival_tags()`**
- **功能**：为 DataFrame 添加"春节标签"列。根据农历"正月初一"定位春节，计算相对周数位置。
- **标签格式**："节前N周"、"春节当周"、"节后N周"

**`generate_lunar_table_with_tags()`**
- **功能**：生成指定年份范围内的农历日期对照表，使用 ZhDate 库转换阳历→农历，再生成春节标签。

**`macro_create_festival_pivot()`**
- **功能**：生成基于春节标签的透视表。从工作表读取日期和数值数据，结合农历日期计算春节相对周数，按标签和年份透视。
- **可复用范围**：宏观行业内所有需要分析春节效应的数据表

**`macro_create_chuxi_pivot()`**
- **功能**：生成基于除夕相对天数的透视表。以除夕当天为"t"，前30天"t-N"至后60天"t+N"为行、年份为列。
- **特殊逻辑**：处理跨年，除夕日期通过农历计算（腊月三十或廿九）

**`macro_create_weekly_pivot()`**
- **功能**：生成按 ISO 周统计的透视表。从工作表提取 ISO 年份和周数，按周数×年份透视。

**`macro_create_yearly_date_scaffold()`**
- **功能**：生成按年分列的日期脚手架（每天一行），每年占用两列（日期+预留数值列）。
- **用途**：为后续数据录入或引用提供日期框架

**`write_df_to_ws()`**
- **功能**：通用 DataFrame 写入工作表方法，支持表头，0值自动替换为None。

### 2.3 银行行业（plugins/bank_plugin.py）

#### 银行整体处理流程（商业银行关键监管指标）
1. 写入基础指标数据（多列日期头，支持单位转换）
2. 应用公式模板（格式转换、环比差值、同比差值等）

#### 核心处理函数

**`bank_write_data()`**
- **功能**：处理各省份不良率数据。从 data_reader 读取多个指标，以第一个指标日期为基准，筛选季度月份（3、6、9、12月）数据，按日期对齐写入。
- **入参**：start_row、start_column、start_date、date_format
- **特殊逻辑**：仅保留季度末月份数据；继承源列格式

**`bank_commercial_write_data()`**
- **功能**：处理商业银行关键监管指标数据。支持多列日期写入（date_columns 参数），支持从指定行读取单位转换因子。
- **入参**：start_row、date_columns、unit_conversion、start_date、date_format
- **特殊逻辑**：多列日期写入（各数据区域独立日期列）；单位转换因子从工作表指定行读取

**`bank_commercial_formula()`**
- **功能**：通用公式写入函数。支持自定义公式模板和占位符替换。
- **支持的占位符**：
  - `{source_column}` → 源列字母
  - `{row}` → 当前行号
  - `{row-N}` → 当前行号偏移 N
  - `{tname}` → 工作表名称（跨表引用）
- **常用公式模板示例**：
  - `={source_column}{row}/100` — 值转为百分比
  - `=IF(ISBLANK({source_column}{row}),"",{source_column}{row}/100)` — 空值保护百分比
  - `={source_column}{row}-{source_column}{row-1}` — 环比差值
  - `={source_column}{row}-{source_column}{row-4}` — 同比差值（季度数据）
  - `=IF(OR(ISBLANK(...),ISBLANK(...)),"",({source_column}{row}-{source_column}{row-4})*100)` — 带空值保护的 BPS 差值

#### 银行特殊格式与公式一览（按 sheet）

| sheet 名称 | 公式类型 | 功能 |
|-----------|---------|------|
| 总资产 | `=col/100` | 值转百分比（T-Y列） |
| 资产增速及增量 | `=IF(ISBLANK,col/100,"")` | 空值保护百分比 |
| 资产增速及增量 | `=tname!col-row - tname!col-(row-1)` | 跨表环比增量 |
| 净息差 | `=(col-row - col-(row-1))*100` | BPS 差值 |
| 净息差 | `=(col-row - col-(row-4))*100` | 同比 BPS（季） |
| 净利润 | `=col-row/col-(row-4)-1` | 同比上年增长（季） |
| 净利润 | `=IF(MONTH(Brow)=3,col-row,col-row-col-(row-1))` | 季度累积逻辑 |
| 不良率 | `=(col-row - col-(row-4))*100` | 同比 BPS（半年数据） |
| 拨备覆盖率 | `=col-row-col-(row-1)` | 环比差值（pct单位） |
| 资本 | `=col/100` | 值转百分比 |

### 2.4 医疗健康行业（plugins/medical_plugin.py）

#### 医疗整体处理流程
1. 纵向源数据 → 横向目标数据转换（日期为列，指标为行）
2. 根据月度数据写入季度列头
3. 写入同比和环比公式
4. 合并季度数据（将同季度月份值合并到季度起始列）

#### 数据格式转换
```
源格式（纵向）：         目标格式（横向）：
日期        | 指标值      A列: 指标名称 | B列起: 2020.1 | 2020.2 | ...
2020-01-01 | 6.36        第4行: 国内医疗健康融资 | 6.36 | 2.56 | ...
2020-02-01 | 2.56
```

#### 核心处理函数

**`medical_write_data()`**
- **功能**：横向写入医疗数据。从所有指标源数据提取统一时间周期序列，将时间周期作为列头写入，各指标数据按日期对号写入对应列。
- **特殊逻辑**：自动解除合并单元格后写入；合并所有指标日期的并集作为统一列头

**`medical_write_quarter()`**
- **功能**：自动判断季度边界，写入合并季度列头（如 2025Q1）。
- **特殊逻辑**：根据月份计算季度`(month-1)//3+1`，跨年判断，自动合并单元格

**`medical_write_formula()`**
- **功能**：横向写入公式。从起始列向右遍历写入，直到日期行最后一个非空列。
- **支持的占位符**：`{col}`（当前列）、`{col-N}`（左偏移N列）、`{row}`（当前行）、`{source_row}`（源数据行）
- **常用公式**：
  - 同比：`=IF(OR(ISBLANK({col}{source_row}),ISBLANK({col-12}{source_row})),"",({col}{source_row}-{col-12}{source_row})/{col-12}{source_row})`
  - 环比：`=IF(OR(ISBLANK({col}{source_row}),ISBLANK({col-1}{source_row})),"",({col}{source_row}-{col-1}{source_row})/{col-1}{source_row})`

**`medical_merge_quarter_data()`**
- **功能**：将同季度内各月份的数据合并到该季度起始列，清理其他列。
- **特殊逻辑**：优先取季度末月份值，找不到则取最后一列有值的；处理合并单元格的只读保护

### 2.5 电子行业（plugins/electronics_plugin.py）

#### 电子整体处理流程
1. 分区域（section）写入各存储产品价格数据（每个区域独立日期列）
2. 更新图表引用范围为最近两年

#### 核心处理函数

**`electronics_write_sheet()`**
- **功能**：写入多区域工作表。每个区域具有独立的日期列和不同的指标集合，支持日度和周度数据。
- **特殊逻辑**：按 AA 代码去重加载数据，合并所有指标日期并集，按降序排列（最新在前）；各区域独立日期列

**`electronics_update_chart_ranges()`**
- **功能**：更新 Excel 图表数据引用范围，自动截断为最近两年数据。

### 2.6 公用事业行业（plugins/public_plugin.py）

#### 公用事业整体处理流程
1. 写入基础指标数据（日期降序排列，支持单位转换）
2. 写入电力报表的动态标题（自动计算当前月份和上月份）
3. 写入 T+3 滚动报表的表头日期

#### 核心处理函数

**`public_write_data()`**
- **功能**：处理公用事业基础指标数据，写入目标工作表。支持从指定行读取单位转换因子。
- **特殊逻辑**：日期按**降序**排列（最新在前），与航空/银行的升序不同

**`public_elec_write_title()`**
- **功能**：写入电力报表的动态标题。根据最新日期自动计算并格式化主标题（当月）和副标题（上月）。
- **特殊逻辑**：2月份标题显示为"1-2"（合并1-2月数据）；上年12月→本年1月的年份切换

**`public_t3_write_header()`**
- **功能**：写入 T+3（三日滚动）报表的表头日期。根据最新日期和其下一行日期写入两个表头单元格。
